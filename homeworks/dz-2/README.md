# ДЗ-2. Интеграция каталога и склада

## Постановка задачи

Каталог товаров должен получать сведения о доступности товара со склада, но не должен синхронно зависеть от склада при каждом пользовательском запросе.

## Решение

Использовать асинхронную публикацию событий:

1. Warehouse Service изменяет остаток.
2. Сервис публикует `StockChanged`.
3. Catalog Service обновляет локальную read-модель.
4. Пользователь читает каталог без обращения к Warehouse Service.

Это ДЗ намеренно содержит только Markdown и PlantUML. Параметры `structurizr: false` и `openapi: false` заданы в `homework.yml`.
